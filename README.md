# dotfiles

Personal dotfiles managed with [yadm](https://yadm.io/), supporting Arch Linux and macOS.

## Setup

```bash
# Clone
yadm clone git@gitlab.com:stivce96/dotfiles.git

# Bootstrap (installs packages, configs, home dirs)
yadm bootstrap
```

## Bootstrap flow

The bootstrap script runs in this order:
1. `common/` — directories, fzf integration
2. `linux/` — distro detection
3. `linux/arch/` or `linux/ubuntu/` — distro-specific setup

## Package installation

- **pacman**: core CLI tools + Hyprland ecosystem
- **AUR**: via `yay` or `paru` (auto-detected)
- **Homebrew**: on macOS

## Dotfile structure

- `.config/` — application configs (yadm auto-symlinks to `~/.config`)
- `.themes##distro.arch/` — Arch-specific GTK/icon themes
- `.local/` — local binaries and scripts

## Adding new configs

```bash
yadm add ~/.config/foo/bar
yadm commit -m "add foo config"
yadm push
```

## Notes

- Colors in `~/.config/hypr/colors.conf` are generated by matugen — not tracked. Regenerate with:
  ```bash
  matugen image ~/Pictures/wallpapers/your-wallpaper.png
  ```
- Shell history (`~/.histfile`) is not tracked
- No SSH keys or machine-specific config included
